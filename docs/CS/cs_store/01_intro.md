# 电脑存储空间管理

---

## 0 前言：为什么需要管理存储空间？
对于高性能笔记本（本人简陋的 Lenovo Legion Y9000P）而言，磁盘的读写速度直接影响着系统响应和开发效率。随着开发环境、GIS 大数据量文件以及各种缓存的堆积，磁盘空间会迅速缩减。

有效的存储管理还是挺重要的
* **保持系统性能**：固态硬盘（SSD）在容量接近满载时，读写性能会大幅下降。
* **数据安全**：合理的分区与备份策略可以防止系统崩溃导致的数据丢失。
* **工作流优化**：清晰的文件组织结构能让我们在处理复杂的 SRTP 项目或 FDS 练习时事半功倍。



## 1 核心内容
本手册旨在记录一套适合开发者和 GIS 学生的存储管理方案，主要涵盖以下几个模块：

### 1.1 磁盘分区与规划
* **系统盘（C盘）**：仅存放操作系统和核心软件，定期清理 `AppData` 与临时文件。
* **开发盘（D盘/E盘）**：统一存放项目代码、Conda 环境以及本地数据库。
* **数据盘**：专门用于存放大型遥感影像、矢量数据等 GIS 素材。

### 1.2 环境与缓存深度清理
* **编译产物**：定期处理 C/C++ 编译产生的 `.exe` 或中间文件。

### 1.3 自动化与工具推荐
* **SpaceSniffer/WizTree**：可视化分析磁盘占用，精准定位“空间杀手”。
* **qpdf & ExifTool**：在压缩文档体积的同时，保护文件元数据隐私。
* **符号链接 (Symbolic Link)**：将占用巨大的文件夹迁移到非系统盘的进阶技巧。



## 管理原则
> **"Data is clutter until it is organized."**
> 
> * **定期审计**：每月进行一次大扫描。
> * **云端同步**：重要的代码通过 GitHub 管理，减少本地冗余。
> * **冷热分离**：高频使用的项目放在 SSD，存档的旧数据移动至云端或移动硬盘。
