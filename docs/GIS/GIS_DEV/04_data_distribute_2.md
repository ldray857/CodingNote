# 四、空间数据发布与服务（下）

在 WebGIS 环境下，空间数据的发布（如 GeoServer 发布的 WMS/WFS 服务）本质上是基于计算机网络协议的数据交换。理解网络底层逻辑是解决“地图加载慢”、“接口请求失败”等问题的核心。

---





## 4. TCP 三次握手 (Three-way Handshake)

当你请求一张地图图片时，浏览器必须先通过 TCP 协议与服务器建立“可靠”连接。



1.  **第一次握手 (SYN)**：
    * **客户端**：发送同步序列号 `SYN`。
    * *意图*：“你好，我想和你建立连接。”
2.  **第二次握手 (SYN + ACK)**：
    * **服务器**：发送 `SYN` 和确认号 `ACK`。
    * *意图*：“收到请求了，我也准备好了，你确认一下？”
3.  **第三次握手 (ACK)**：
    * **客户端**：发送 `ACK`。
    * *意图*：“确认完毕，我们开始传数据吧！”

**为什么 GIS 开发者要关注这个？**
在高并发请求地图瓦片时，如果握手时间过长，会导致地图出现“白块”。优化 TCP 连接（如使用 HTTP/2 或 HTTP/3）是提升地图流畅度的关键。
