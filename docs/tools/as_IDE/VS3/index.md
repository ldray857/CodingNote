# VS、VS code and VS installer
*引言：* *关于为什么要介绍vs code——我相信大部分人都知道vs code或者听过其大名，但常常对于VS、VS code 、 VS installer三者的定义模糊。希望通过这个文档，你对三者有更新的体会。*

---

## 1. 背景公司介绍
**微软（Microsoft）：** 在早年间，微软的工具（如 Visual Studio）主要服务于 Windows 平台的开发者。但自从 萨提亚·纳德拉（Satya Nadella） 接任 CEO 后，微软的策略发生了巨大的变化，口号变成了“微软爱开源”。

VS Code 是这一转型的代表作。微软意识到开发者不再只用 Windows，很多人在 Mac 或 Linux 上写代码。于是他们基于开源技术（Electron）打造了 VS Code，并将其代码在 GitHub 上开源。

## 2. 产品基本介绍
## 2.1 Visual Studio Code (VS Code) 

**定位**：一款开源、跨平台的**现代化代码编辑器**。
**核心特点**：
* **轻量与极速**：启动极快，内存占用相对较小。
* **插件库齐全**：通过安装扩展，它可以完成各种内容的轻量级开发。
* **跨平台**：在 Windows、macOS 和 Linux 上体验完全一致。  

对于普通的coding来说，我们常常使用vs code，关于我对vs code的一些使用经验，大家可以调到我的这个部分查看 [VScode入门](VScode/)

## 2. Visual Studio (VS)
**定位**：被誉为“宇宙第一 IDE”的**全功能集成开发环境**。
**核心特点**：
* **重型IDE**：内置了强大的编译器、调试器、性能分析器和图形化界面设计器。
* **深度集成**：对 .NET 和 C++ 的支持是工业级的，适合管理拥有数千个文件的复杂项目。
* **Windows 原生**：虽然有 Mac 版，但其最强功能始终根植于 Windows。
* **适用场景**：开发大型 Windows 桌面程序、专业游戏开发（如使用 Unreal Engine）、复杂的 C++ 算法研究。

## 3. Visual Studio Installer —— 幕后管家

**定位**：用于安装、管理和修改 Visual Studio 的官方工具。
**主要功能**：
* **按需取用**：Visual Studio 体积庞大，Installer 允许你通过勾选“工作负载”（如“使用 C++ 的桌面开发”）来只安装需要的组件，节省磁盘空间。
* **一键更新**：管理 VS 的版本升级和修复。    

它服务于 Visual Studio；而 VS Code 通常是独立安装的，不通过它管理。

<br/>

### 三者对比

| 特性 | VS Code | Visual Studio | VS Installer |
| --- | --- | --- | --- |
| **工具类型** | 代码编辑器 | 集成开发环境 (IDE) | 安装/管理工具 |
| **资源占用** | 较低 | 较高 | 极低 |
| **上手难度** | 简单，即装即用 | 较复杂，配置项多 | 简单 |
| **核心优势** | 灵活、生态丰富 | 功能极其完备、调试强大 | 模块化安装、易于维护 |
